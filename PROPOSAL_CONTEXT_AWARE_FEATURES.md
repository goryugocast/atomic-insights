# Atomic Insights 2.0: 文脈（コンテキスト）認識による発見機能

## 提案概要
既存の「Atomic Insights」プラグインを拡張し、グラフ構造だけでなく、ファイル名やメタデータなどの「文脈」を考慮した発見機能を提供する提案です。

## 提案される変更

### [ハイブリッド・スコアリング・エンジン]
グラフスコア（Adamic Adar）とメタデータスコアを組み合わせた新しいスコアリングシステム。

メタデータスコアの構成要素:

1.  **絵文字マッチ（感情/種類の文脈）**
    *   ファイル名から絵文字（例：🥁, 📚）を抽出。
    *   ソースとターゲットが同じ絵文字を共有している場合：**+ボーナスポイント**。
    *   これらは「シリーズ」や「特定ジャンル」のノートであることが多く、関連性が高い。

2.  **YAMLメタデータマッチ（トピックの文脈）**
    *   設定でユーザー定義可能なキー（デフォルト：`project`, `context`, `tags`）。
    *   例：`project: ks` が一致する場合：**+ボーナスポイント**。

3.  **時間的近接性（時間の文脈）**
    *   ファイル名から日付（`YYMMDD` または `YYYY-MM-DD`）を抽出。
    *   日付の差分  = |Date_{source} - Date_{target}|$ を計算。
    *   直近（1週間以内など）のノートには高いスコアを与える。

### [設定の更新]
*   **メタデータスコアリングの切り替え**: オン/オフ。
*   **メタデータキー**: マッチさせるYAMLキーのリスト。
*   **重み付け**: 「グラフ構造」重視か「文脈」重視かのバランス調整。

### [UIの変更]
*   選出理由の視覚的インジケータ（例：「🔗 グラフ」、「📅 日付」、「🥁 絵文字」）。
